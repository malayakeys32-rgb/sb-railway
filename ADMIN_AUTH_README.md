# Admin Authentication System\n\n## Overview\n\nComprehensive admin login system for Sentinel Black with enterprise-grade security features:\n\n✅ **Admin-Only Login** — Restricted to authorized admin accounts  \n✅ **Multi-Factor Authentication (MFA)** — Optional 6-digit code verification  \n✅ **Email Domain Restriction** — Only whitelisted domains can access admin accounts  \n✅ **Password Reset** — Secure token-based password recovery (24-hour expiry)  \n✅ **Role-Based Access** — ADMIN, REPORTER, VIEWER roles with fine-grained permissions  \n✅ **Audit Logging** — Complete forensic log of all admin actions  \n✅ **Admin Dashboard** — Real-time statistics, user management, and audit log viewer  \n\n---\n\n## Setup\n\n### Environment Variables\n\nAdd these to your `.env` file:\n\n```env\n# JWT configuration (required)\nJWT_SECRET=your_super_secret_jwt_key_here\n\n# Admin email domain(s) allowed (comma-separated)\nADMIN_EMAIL_DOMAINS=admin.local,organization.local\n\n# API base URL for frontend\nNEXT_PUBLIC_API_URL=https://sb-railway-production.up.railway.app\n```\n\n### Database Setup\n\nThe Prisma schema already includes:\n- User table with `role` field (ADMIN, REPORTER, VIEWER)\n- `mfaEnabled` boolean flag per user\n- AuditLog table for forensic tracking\n\nRun migrations if needed:\n```bash\ncd sentinel-black-railway-deploy\nnpm run prisma:migrate\n```\n\n---\n\n## API Endpoints\n\n### Admin Login\n\n**POST** `/auth/admin/login`\n\nRequest:\n```json\n{\n  \"email\": \"admin@organization.local\",\n  \"password\": \"SecurePassword123!\"\n}\n```\n\nResponse (without MFA):\n```json\n{\n  \"token\": \"eyJhbGciOiJIUzI1NiIs...\",\n  \"user\": {\n    \"id\": \"user_123\",\n    \"email\": \"admin@organization.local\",\n    \"name\": \"Admin User\",\n    \"role\": \"ADMIN\",\n    \"mfaEnabled\": false,\n    \"createdAt\": \"2026-07-15T00:00:00Z\"\n  }\n}\n```\n\nResponse (with MFA enabled):\n```json\n{\n  \"requiresMFA\": true,\n  \"userId\": \"user_123\",\n  \"message\": \"MFA code sent to email\"\n}\n```\n\n**Validation:**\n- Email domain must be in `ADMIN_EMAIL_DOMAINS`\n- User must have `role: ADMIN`\n- Password must match bcrypt hash\n\n---\n\n### Verify MFA Code\n\n**POST** `/auth/admin/verify-mfa`\n\nRequest:\n```json\n{\n  \"userId\": \"user_123\",\n  \"mfaCode\": \"123456\"\n}\n```\n\n**Rules:**\n- Code expires after 10 minutes\n- Maximum 3 failed attempts before lockout\n- Code is single-use\n\n---\n\n### Request Password Reset\n\n**POST** `/auth/admin/forgot-password`\n\nRequest:\n```json\n{\n  \"email\": \"admin@organization.local\"\n}\n```\n\nResponse:\n```json\n{\n  \"message\": \"If account exists, reset link has been sent\"\n}\n```\n\n**Note:** Always returns success (doesn't reveal if email exists)\n\n---\n\n### Reset Password\n\n**POST** `/auth/admin/reset-password`\n\nRequest:\n```json\n{\n  \"resetToken\": \"abc123def456...\",\n  \"email\": \"admin@organization.local\",\n  \"newPassword\": \"NewSecurePassword123!\"\n}\n```\n\n**Rules:**\n- Token expires after 24 hours\n- Password must be at least 12 characters\n- Single-use token\n\n---\n\n### Admin Dashboard\n\n**GET** `/auth/admin/dashboard`\n\nHeaders:\n```\nAuthorization: Bearer <jwt_token>\n```\n\nResponse:\n```json\n{\n  \"stats\": {\n    \"totalUsers\": 45,\n    \"totalIncidents\": 128,\n    \"totalEvidence\": 342\n  },\n  \"recentAuditLogs\": [\n    {\n      \"id\": \"log_123\",\n      \"action\": \"ADMIN_LOGIN\",\n      \"resource\": \"admin_auth\",\n      \"userId\": \"user_123\",\n      \"user\": {\n        \"email\": \"admin@organization.local\",\n        \"name\": \"Admin User\"\n      },\n      \"createdAt\": \"2026-07-15T12:34:56Z\"\n    }\n  ]\n}\n```\n\n---\n\n### List All Users\n\n**GET** `/auth/admin/users`\n\nHeaders:\n```\nAuthorization: Bearer <jwt_token>\n```\n\nResponse:\n```json\n[\n  {\n    \"id\": \"user_123\",\n    \"email\": \"admin@organization.local\",\n    \"name\": \"Admin User\",\n    \"role\": \"ADMIN\",\n    \"mfaEnabled\": true,\n    \"createdAt\": \"2026-07-15T00:00:00Z\"\n  }\n]\n```\n\n---\n\n### Update User\n\n**PATCH** `/auth/admin/users/:userId`\n\nHeaders:\n```\nAuthorization: Bearer <jwt_token>\n```\n\nRequest:\n```json\n{\n  \"role\": \"REPORTER\",\n  \"mfaEnabled\": true\n}\n```\n\n---\n\n### Delete User\n\n**DELETE** `/auth/admin/users/:userId`\n\nHeaders:\n```\nAuthorization: Bearer <jwt_token>\n```\n\n**Note:** Cannot delete your own account\n\n---\n\n## Frontend Routes\n\n### Admin Login Page\n- **URL:** `/admin/login`\n- **Features:** Login form → MFA code entry → Dashboard\n- **Password Reset:** Forgot password flow with recovery link\n\n### Admin Dashboard\n- **URL:** `/admin/dashboard`\n- **Tabs:**\n  - **Overview:** System statistics (users, incidents, evidence)\n  - **Users:** User management table with role/MFA status\n  - **Logs:** Forensic audit log viewer\n\n---\n\n## Security Considerations\n\n1. **JWT Expiry:** Admin tokens expire after 1 hour (configurable in code)\n2. **MFA Codes:** 6-digit codes valid for 10 minutes, 3 attempts max\n3. **Password Reset Tokens:** Valid for 24 hours, single-use\n4. **Email Domain Validation:** Only whitelisted domains can create/login as admins\n5. **Audit Logging:** All admin actions logged with timestamp, user, IP (when available)\n6. **No Brute Force Protection:** Consider adding rate limiting in production\n\n### Production Checklist\n\n- [ ] Set strong `JWT_SECRET` (min 32 characters)\n- [ ] Configure correct `ADMIN_EMAIL_DOMAINS`\n- [ ] Move MFA codes to Redis (currently in-memory)\n- [ ] Move password reset tokens to database\n- [ ] Add email service for MFA/reset emails\n- [ ] Add rate limiting on auth endpoints\n- [ ] Enable HTTPS for all routes\n- [ ] Configure CORS properly for your domain\n- [ ] Regular security audits of audit logs\n- [ ] Implement session timeout\n\n---\n\n## Audit Log Events\n\nCommon audit actions:\n\n| Action | Details |\n|--------|----------|\n| `ADMIN_LOGIN` | Successful admin login |\n| `ADMIN_LOGIN_FAILED` | Failed login attempt |\n| `ADMIN_LOGIN_MFA_SUCCESS` | Successful MFA verification |\n| `MFA_VERIFICATION_FAILED` | Failed MFA code |\n| `PASSWORD_RESET_REQUESTED` | Password reset initiated |\n| `PASSWORD_RESET_SUCCESS` | Password successfully reset |\n| `USER_UPDATED` | User role/settings changed |\n| `USER_DELETED` | User account deactivated |\n\n---\n\n## Example Usage\n\n### Create Admin User (via database)\n\n```sql\nINSERT INTO users (id, email, password, name, role, \"mfaEnabled\", \"createdAt\", \"updatedAt\")\nVALUES (\n  'cuid_123',\n  'admin@organization.local',\n  '$2a$12$...bcrypt_hash...',  -- bcryptjs hash of password\n  'Admin User',\n  'ADMIN',\n  true,\n  NOW(),\n  NOW()\n);\n```\n\n### Login Flow (Frontend)\n\n```javascript\n// 1. Submit credentials\nconst res = await fetch('/auth/admin/login', {\n  method: 'POST',\n  body: JSON.stringify({ email, password })\n});\n\n// 2. If MFA required, prompt for code\nif (res.data.requiresMFA) {\n  const mfaRes = await fetch('/auth/admin/verify-mfa', {\n    method: 'POST',\n    body: JSON.stringify({ userId, mfaCode })\n  });\n  localStorage.setItem('token', mfaRes.data.token);\n}\n\n// 3. Use token for authenticated requests\nfetch('/auth/admin/dashboard', {\n  headers: { Authorization: `Bearer ${token}` }\n});\n```\n\n---\n\n## Troubleshooting\n\n### \"Invalid admin email domain\"\n- Check `ADMIN_EMAIL_DOMAINS` environment variable\n- Email must match one of the configured domains\n\n### \"Admin access denied\"\n- User account must have `role: ADMIN` in database\n- User created as REPORTER cannot access admin endpoints\n\n### MFA code not working\n- Code expires after 10 minutes\n- Limited to 3 failed attempts\n- Check server logs for generated code (development only)\n\n### Password reset link expired\n- Token valid for 24 hours only\n- Request a new reset link\n\n---\n\n## Next Steps\n\n1.
